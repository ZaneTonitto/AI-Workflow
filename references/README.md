# References

Reference documentation that agents can consult during their work.

## What Goes Here?

Any material that provides context an agent needs to make good decisions:

- **Coding Standards** — Language-specific style guides and conventions.
- **Architecture Docs** — System design, component diagrams, data flow.
- **API Specifications** — OpenAPI specs, GraphQL schemas, internal API docs.
- **Style Guides** — UI/UX patterns, naming conventions, commit message formats.
- **Runbooks** — Operational procedures, deployment checklists.
- **Domain Knowledge** — Business rules, glossaries, domain models.

## Conventions

- Organize by category using subdirectories (e.g., `references/coding-standards/`, `references/architecture/`).
- Use Markdown for prose, YAML/JSON for structured data.
- Keep files focused — one topic per file.
- Include a brief header comment explaining when/why an agent should consult the file.
